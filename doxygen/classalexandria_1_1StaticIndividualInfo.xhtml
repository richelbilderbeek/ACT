void alexandria::StaticIndividualInfo::setOutputFiles ( const char * xvgconv,
const std::vector< std::string > & paramClass,
const char * xvgepot )

Set the output file names.

The parameter values are split over a number of files in order to make it easier to visualize the results. The parameter classes should therefore match the parameter names. E.g. a class could be alpha, another zeta.

Parameters
[in]xvgconvThe parameter convergence base name
[in]paramClassThe parameter classes (e.g. zeta, alpha)
[in]xvgepotThe base filename to print the value

void alexandria::StaticIndividualInfo::updateForceField ( MsgHandler * msghandler,
const std::set< int > & changed,
const std::vector< double > & bases )

Copy the Force Field parameters to the ForceField structure.

Parameters
[in]msghandlerWrite (debugging) info and handle errors
[in]changedList over the parameters that have changed. If empty, all parameters will be updated. The numbers are indices in the internal optIndex structure.
[in]basesThe parameters in the current genome. If size equals zero the routine will do nothing and return.
Exceptions
iflength of bases > 0 and the length does not match the known number of parameters.
